## 📋 현재 상황 분석

### Canvas Index 페이지 구조 (이미지 분석)

**레이아웃**:
- 헤더: "Projects" 제목 + "+ Create" 버튼
- 섹션별 분류: Current Projects / Past Projects (Upcoming, Unpublished도 존재)
- 테이블 형식:
  - 컬럼: Project Name | STEP 1 | STEP 2 | STEP 3 | ...
  - 각 프로젝트는 한 행으로 표시

**프로젝트 행 구조**:
- **Project Name 셀**:
  - 프로젝트 이름 (굵은 글씨)
  - 날짜 정보: "Start - - Due MMM D, H:mm / End -"
  - 액션 버튼: 보기/수정/삭제 (추정)
- **STEP 셀**:
  - 아이콘: 💬 (Slack), ⋮ (More 메뉴)
  - 마감일: "Due MMM D, H:mm"
  - 상태 뱃지: Past/Current/Upcoming
  - 교수용: Needs Grading 뱃지 (연필 아이콘)
  - 학생용: 제출 완료/미제출 뱃지

**과목(코스) 정보**:
- 이미지에서 **과목 정보가 별도로 표시되지 않음**
- 모든 프로젝트가 같은 코스에 속한 것으로 보임
- Canvas는 `/courses/:course_id/projects` 경로로 특정 코스의 프로젝트만 보여줌

### LTI Index 페이지 현재 구조

**레이아웃**:
- 헤더: "Projects" 제목 + "+ Create" 버튼 ✅
- 섹션별 분류: Current / Upcoming / Past / Unpublished ✅
- 테이블 형식: ✅

**프로젝트 행 구조**:
- Project Name 셀: ✅
  - 프로젝트 이름 ✅
  - 날짜 정보 ✅
  - 액션 버튼 ✅
- STEP 셀: ✅
  - 아이콘: 💬, ⋮ ✅
  - 마감일 ✅
  - 상태 뱃지 ✅
  - 교수/학생 분기 통계 ✅

**과목(코스) 정보**:
- 현재: **표시되지 않음**
- LtiContext에 `context_title` 저장되어 있음
- Canvas API로 코스 정보 조회 가능

## 🔍 과목(코스) 정보 표시 가능 여부 분석

### 현재 데이터 구조

#### LtiContext 모델
```ruby
# db/schema.rb
create_table "lti_contexts" do |t|
  t.string "context_id"        # LTI Context ID
  t.string "context_type"       # "Course"
  t.string "context_title"      # 코스 제목 ✅ 저장됨
  t.integer "canvas_course_id"  # Canvas Course ID ✅ 저장됨
  # ...
end
```

#### LTI Launch 시 저장되는 정보
```ruby
# app/controllers/lti/launch_controller.rb
context_title: context["title"]  # LTI Claims에서 추출
canvas_course_id: custom_course_id || canvas_course_id_claim
```

### Canvas API로 코스 정보 조회 가능 여부

#### 옵션 1: LtiContext에 저장된 정보 사용
- ✅ **장점**: API 호출 불필요, 빠름
- ✅ **데이터**: `context_title` 필드에 코스 제목 저장됨
- ⚠️ **한계**: 코스 정보가 변경되면 동기화 필요

#### 옵션 2: Canvas API로 실시간 조회
- ✅ **장점**: 최신 정보 보장
- ⚠️ **단점**: API 호출 필요 (성능)
- ✅ **가능**: Canvas API `/api/v1/courses/:id` 엔드포인트 사용

#### 옵션 3: 여러 코스 지원
- ⚠️ **현재 구조**: 하나의 LtiContext(코스)에 속한 프로젝트만 표시
- ⚠️ **한계**: LTI Launch는 특정 코스에서만 실행됨
- ✅ **대안**: 여러 코스의 프로젝트를 보려면 아키텍처 변경 필요

## 💡 주요 결정 사항

### 1. 과목 정보 표시
**결정**: 헤더에 현재 코스 이름 표시 (옵션 B)
**이유**: 
- Canvas는 URL에 코스 정보가 있지만 LTI는 없음
- 사용자가 현재 코스를 명확히 알 수 있음
- 같은 코스 내 프로젝트만 보여주므로 행별 표시는 불필요

### 2. 여러 코스 지원
**결정**: 현재는 지원하지 않음
**이유**:
- LTI Launch는 특정 코스에서만 실행됨
- 여러 코스 지원하려면 아키텍처 변경 필요 (별도 작업)
- **Canvas도 같은 코스 내 프로젝트만 보여줌** (이미지 확인)

**API 통신으로 가능 여부**:
- ✅ **가능**: Canvas API `/api/v1/courses`로 여러 코스 조회 가능
- ⚠️ **한계**: LTI Launch는 특정 코스에서만 실행되므로, 다른 코스의 프로젝트를 보려면:
  - 옵션 A: 각 코스에서 LTI Launch 실행 (현재 방식)
  - 옵션 B: 아키텍처 변경 (모든 코스의 프로젝트 통합 보기)
    - 사용자별로 여러 LtiContext 조회
    - 각 코스별로 Canvas API 호출
    - 성능 이슈 가능

**결론**: 현재는 같은 코스 내 프로젝트만 보여주는 것이 Canvas와 동일하므로, 과목 정보는 헤더에만 표시하는 것으로 충분

### 3. Slack 아이콘
**결정**: 표시하지 않음 (의도적 제외)
**이유**: Slack 기능이 없으므로 아이콘만 표시하는 것은 혼란

## 🐛 비즈니스 로직 수정 (2026-01-10)

### 문제: 하나의 프로젝트가 Current와 Unpublished에 동시에 표시됨

**원인**:
- `classify_projects` 메서드에서 각 카테고리를 독립적으로 select
- Unpublished이면서 active step이 있으면 current와 unpublished 둘 다에 포함됨
- Canvas의 로직과 다름

**Canvas hy_projects 로직 (index.jsx)**:
1. **먼저 Published/Unpublished 분리**:
   ```javascript
   const publishedProjects = projects.filter(project => project.published !== false)
   ```
2. **Published 프로젝트만으로 Current/Upcoming/Past 분류**:
   - Upcoming: 모든 스텝이 시작 전
   - Current: Upcoming도 Past도 아닌 것
   - Past: 모든 스텝이 마감일 지남
3. **Unpublished 프로젝트는 별도 관리**:
   ```javascript
   const unpublishedProjects = projects.filter(project => project.published === false)
   ```

**우선순위**:
1. **Unpublished 프로젝트는 unpublished에만 포함** (최우선)
2. **Published 프로젝트만 current/upcoming/past에 포함**
3. **Current/Upcoming/Past는 상호 배타적** (하나에만 포함)

**수정 내용**:
- `app/services/project_service.rb`의 `classify_projects` 메서드 수정
- Canvas 로직과 동일하게 구현:
  1. Published/Unpublished 먼저 분리
  2. Published 프로젝트만으로 current/upcoming/past 분류
  3. Current: Upcoming도 Past도 아닌 Published 프로젝트
  4. Unpublished 프로젝트는 별도 섹션에만 표시
  5. **학생은 Unpublished 프로젝트를 볼 수 없음** (Canvas 로직)
- `projects_with_statistics` 메서드에서 user_role 확인하여 is_student 전달

**수정 파일**: `app/services/project_service.rb`

**변경 사항**:
